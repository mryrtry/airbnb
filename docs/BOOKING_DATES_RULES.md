# Правила работы с датами заявок на бронирование

## Модель

- У каждой заявки на бронирование есть **дата въезда** (`checkInDate`) и **дата выезда** (`checkOutDate`) — тип `LocalDate`, в API передаются в формате ISO (`yyyy-MM-dd`).
- Интервал проживания: **[checkInDate, checkOutDate)** — в дату выезда гость уже освобождает жильё, в эту же дату может заехать следующий (два человека не занимают одни и те же даты в одном объявлении).

## Бизнес-правила

1. **Дата выезда строго позже даты въезда.**  
   При создании заявки проверяется `checkOutDate > checkInDate`. Иначе 400 (BOOKING_CHECK_OUT_AFTER_CHECK_IN).

2. **Один гость — не более одной заявки с пересекающимися датами по одному объявлению.**  
   При создании проверяется: у текущего пользователя нет другой заявки по этому же `listingId` со статусом APPLIED, APPROVED или CHECKED_IN и с пересекающимися датами. Пересечение: `checkIn1 < checkOut2 && checkOut1 > checkIn2`. Иначе 400 (BOOKING_GUEST_OVERLAPPING_DATES).

3. **Несколько заявок на одни и те же даты по одному объявлению допускаются.**  
   Разные гости могут подать заявки на одни и те же даты; все они имеют статус APPLIED до решения владельца.

4. **При одобрении одной заявки остальные с пересекающимися датами отклоняются.**  
   При переводе заявки в APPROVED все другие заявки (APPLIED) по этому же объявлению с пересекающимися датами автоматически переводятся в REJECTED. Подавшим эти заявки отправляется уведомление **BOOKING_REJECTED_ANOTHER_APPROVED** (текст: заявка отклонена, на эти даты одобрена заявка другого гостя).

5. **Нельзя одобрить заявку, если на эти даты уже одобрена или занята другая бронь.**  
   Перед переводом в APPROVED проверяется: по этому объявлению нет другой брони со статусом APPROVED или CHECKED_IN с пересекающимися датами. Иначе 400 (BOOKING_LISTING_OCCUPIED_DATES).

## API

- **POST /bookings** — тело: `listingId`, `checkInDate`, `checkOutDate` (все обязательны). Валидация по правилам 1 и 2.
- **PUT /bookings/{id}/decide** с `status: APPROVED` — валидация по правилу 5; затем массовое отклонение по правилу 4 и рассылка уведомлений.

## Фильтрация

- **GET /bookings** поддерживает query-параметры `checkInDateFrom` и `checkOutDateTo` (ISO) для фильтрации по датам заезда/выезда (см. `BookingFilter`).
